# 依赖配置

这部分内容涉及附魔的依赖配置。

```yaml
dependencies:
  supports: 11600-12005
  datapacks: [ ]
  plugins:
    - Adyeshach
```

## 节点介绍

| 节点            | 描述                       |
|-----------------|----------------------------|
| `supports`        | 版本支持区间，默认为 `1.16` 至最新版本 |
| `datapacks`| 依赖数据包           |
| `plugins`            | 依赖插件 |

### 版本支持区间

这里需要填写我们要求的一种特殊的版本号表现形式，具体表现为：

- `1.7.10` -> `10710`
- `1.8` -> `10800`
- `1.12` -> `11200`
- `1.16.5` -> `11605`
- `1.20.4` -> `12004`

总的来说就是将原本两位或三位的版本号数字拆成两部分或三部分，去掉中间的 **点**，并分别处理。这里我们以 `1.20.4` 举例：

- `1` 此项只有一位数字，直接保留，不做任何处理
- `20` 此项为两位数字时直接保留，如果是一位则在前加 `0` 来补足成两位数字
- `4` 此项不存在时直接写 `00`，为两位数字时直接保留，如果是一位则在前加 `0` 来补足两位数字

去掉 `1.20.4` 中的 **点**，直接保留第一位数字，将两位数字的第二项保留，并将第三项补为两位数字，就可以得到 `12004`。

:::tip

这是由 [坏黑先生](https://github.com/Bkm016) 发明的版本号表现形式，十分可爱。

:::

当想要表示一个区间时，在两个版本号中间加 `-`，意为区间，例如写成 `11605-12004` 表示支持从 `1.16.5` 到 `1.20.4` 的所有版本，默认为闭区间。

当只写一个版本号时，表示支持该版本以及更新的所有版本，例如写成 `11802` 表示支持 `1.18.2` 版本至最新版本。
